# 왜 이 블로그를 만들었나

프로젝트를 여러 번 다시 볼수록, "왜 이렇게 만들었지?"를 찾는 시간이 길어졌어요.

그래서 이 블로그는 단순한 포트폴리오보다 **기술 판단의 기억 장치**에 가깝게 설계했어요.

## 섹션 구조

- `diary`: 길게 설명하는 실험 기록
- `tweet`: 짧은 메모와 링크
- `review`: 회고와 판단 정리

## 운영 방식

- 주소는 slug 중심으로 유지해요.
- 날짜는 글 메타데이터와 목록에서 보여줘요.
- 글 원본은 Markdown `.md`로 관리해요.
